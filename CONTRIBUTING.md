# Contribution Guidelines

## Branching Strategy
- Use feature branches for new development. 
- Naming convention: `feature/your-feature-name`.
- Always create a branch from the `main` branch.

## Code Style Guidelines
- Follow [Airbnb JavaScript Style Guide](https://github.com/airbnb/javascript).
- Ensure proper formatting and linting are applied before submitting code.

## Pull Request Process
- Submit a pull request against the `main` branch.
- Provide a clear title and description of your changes.
- Ensure that your code passes all tests before submission.

## Team Communication
- Use Slack for real-time communication.
- Weekly meetings are held every Monday at 10 AM UTC.

## Testing Requirements
- Write unit tests for new features and regression tests for bugs.
- Ensure all tests pass before merging.

## Documentation
- Update relevant documentation when introducing new features.
- Ensure all changes are reflected in the README and relevant markdown files.

## استراتيجية الفروع
- استخدم فروع الميزات للتطوير الجديد.  
- نمط التسمية: `feature/اسم ميزتك`.  
- قم دائمًا بإنشاء فرع من الفرع `main`.  

## إرشادات أسلوب التعليمات البرمجية
- اتبع [دليل أسلوب جافا سكريبت الخاص بـ Airbnb](https://github.com/airbnb/javascript).  
- تأكد من تطبيق التنسيق الصحيح وقواعد التحقق قبل تقديم التعليمات البرمجية.  

## عملية طلب السحب
- قدم طلب سحب ضد الفرع `main`.  
- قدم عنوانًا ووصفًا واضحًا للتغييرات التي أجريتها.  
- تأكد من أن التعليمات البرمجية الخاصة بك تمر بجميع الاختبارات قبل الإرسال.  

## الاتصالات في الفريق
- استخدم Slack للتواصل في الوقت الفعلي.  
- تُعقد الاجتماعات الأسبوعية كل يوم إثنين في الساعة 10 صباحًا بتوقيت UTC.  

## متطلبات الاختبار
- اكتب اختبارات وحدات للميزات الجديدة واختبارات تجريبية للأخطاء.  
- تأكد من اجتياز جميع الاختبارات قبل الدمج.  

## الوثائق
- تحدث إلى الوثائق ذات الصلة عند تقديم ميزات جديدة.  
- تأكد من عكس جميع التغييرات في README والملفات Markdown ذات الصلة.
